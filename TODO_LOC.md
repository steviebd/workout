## Deprioritized / Not Recommended

### ~~Consolidate API Routes~~

**Not feasible.** TanStack Start uses file-based routing â€” each file path maps directly to a URL. `workouts.$id.ts`, `workouts.$id.complete.ts`, and `workouts.$id.exercises.ts` cannot be merged into a single file without breaking the routing convention.

### ~~Unified Server/Local DB Repository~~

**Not recommended.** The server layer (Drizzle/D1) and local layer (Dexie/IndexedDB) have fundamentally different implementations:

- Server: Drizzle query builder with SQL joins, transactions, pagination
- Local: Dexie IndexedDB with offline queue operations and sync status tracking

A unified interface would add abstraction complexity for marginal savings (~200â€“300 LOC on a 6,089 LOC codebase). The two layers share a *concept* but not an *implementation* â€” the right abstraction is the existing separation.
